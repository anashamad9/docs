The Apps page is where you browse and connect the external tools your workflows depend on. Once an app is connected, its triggers and actions become available to any workflow node in your workspace — letting you listen for events in Gmail, post to Slack, create Jira issues, or update Salesforce records without leaving Atmet. Every integration is configured once per workspace and then shared across all automations.

Navigate to **Apps** in the sidebar to see the full catalog. Each app card shows the integration name, a short description, its authentication type, and whether it is already connected. Use the search bar to filter by name, or click a category button to narrow the list.

## Available integrations

Atmet currently supports ten integrations across five categories.

<CardGroup cols={2}>
  <Card title="Communication" icon="comments">
    **Gmail**, **Slack**, **Discord** — connect your messaging and email tools to trigger workflows from incoming messages or send notifications as part of a workflow run.
  </Card>
  <Card title="Productivity" icon="check-square">
    **Notion**, **Asana** — automate page creation, task management, and project tracking across your productivity stack.
  </Card>
  <Card title="CRM" icon="address-book">
    **HubSpot**, **Salesforce** — sync contact and deal activity with your CRM so workflows can react to pipeline changes and enrich records automatically.
  </Card>
  <Card title="Developer" icon="code">
    **GitHub**, **Jira** — track pull requests, issues, and sprint milestones, and create or update records directly from workflow outcomes.
  </Card>
</CardGroup>

**Social** — **X (Twitter)** lets you publish posts and respond to mentions from workflow action nodes, using the X developer API.

## Connecting an app

The connection flow depends on the app's authentication type.

<Tabs>
  <Tab title="OAuth 2.0">
    OAuth apps — Gmail, Slack, Notion, GitHub, Jira, Asana, Discord — connect through an authorization redirect in your browser.

    <Steps>
      <Step title="Click Connect">
        On the app card, click **Connect**. You are taken to the app's dedicated setup page.
      </Step>
      <Step title="Authorize in your browser">
        A browser window opens to the app's OAuth consent screen. Sign in to the account you want to link, review the requested permissions, and click **Allow** or **Authorize**.
      </Step>
      <Step title="Return to Atmet">
        After authorizing, you are redirected back to Atmet. The app card updates to show a **Connected** badge. The integration is now active for the entire workspace.
      </Step>
    </Steps>
  </Tab>
  <Tab title="API Key">
    API key apps — HubSpot, Salesforce, X (Twitter) — require you to generate a key in the external tool's developer settings and paste it into Atmet.

    <Steps>
      <Step title="Generate your API key">
        Open the external app's settings or developer portal. For HubSpot, create a private app token. For Salesforce, create an API-enabled integration user key. For X, create an app in the developer portal and generate a key and secret.
      </Step>
      <Step title="Paste the key into Atmet">
        On the app's setup page in Atmet, paste the key into the API key field.
      </Step>
      <Step title="Test the connection">
        Click **Test** to verify that Atmet can reach the external API with the provided key. A successful test confirms the credentials are valid.
      </Step>
      <Step title="Save">
        Click **Save** to activate the integration. The app card shows a **Connected** badge once saved.
      </Step>
    </Steps>
  </Tab>
</Tabs>

<Tip>
  Always test your connection before attaching the app to a production workflow. A failed test usually means the API key has insufficient permissions or the OAuth token has expired. Re-authorize or regenerate the key, then test again before saving.
</Tip>

## Managing connected apps

Click **Manage** on any connected app card to open its settings page. From there you can review the active permissions (OAuth scopes), rotate or replace credentials, and disconnect the app if it is no longer needed.

<Warning>
  Disconnecting an app immediately invalidates its triggers and actions across all workflows that depend on it. Workflows referencing a disconnected app will fail when they next run. Reconnect the app or update the affected workflow nodes before disconnecting.
</Warning>
