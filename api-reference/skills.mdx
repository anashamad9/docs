The Skills API lets you build and manage the reusable AI capabilities that power your Atmet workspace. A skill encapsulates a discrete capability — an action to perform, a trigger to listen for, a tool to invoke, or an agent to reason with — and can be attached to any chat or automation to extend what Atmet can do. You can filter the skill list by type, status, or name, making it easy to find and compose the right building blocks for your workflows. Every request must carry your API key and workspace identifier in the request headers.

---

## Authentication

All endpoints require the following headers:

| Header | Description |
|---|---|
| `Authorization` | `Bearer <key>` — your Atmet API key |
| `X-Workspace-Id` | The unique identifier of your workspace |

---

## The Skill object

Every endpoint that returns a skill yields the following fields.

<ResponseField name="id" type="string" required>
  Unique identifier for the skill (UUID).
</ResponseField>

<ResponseField name="name" type="string" required>
  Human-readable name for the skill.
</ResponseField>

<ResponseField name="description" type="string">
  Optional explanation of what the skill does and when to use it.
</ResponseField>

<ResponseField name="type" type="string" required>
  The capability category. One of `action`, `trigger`, `tool`, or `agent`.
</ResponseField>

<ResponseField name="status" type="string" required>
  Whether the skill is usable. One of `active` or `inactive`.
</ResponseField>

<ResponseField name="definition" type="object">
  Optional configuration object for the skill. Use this to supply structured parameters your skill implementation requires (e.g. API endpoint, model settings). The shape is freeform — use whatever keys your skill expects.
</ResponseField>

<ResponseField name="created_by" type="string" required>
  UUID of the user who created the skill.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the skill was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the most recent update.
</ResponseField>

<Note>
  **Skill types map to UI categories:**
  - `action` — a discrete operation the assistant can perform (e.g. send an email, create a record)
  - `trigger` — an event that starts an automation (e.g. a form submission, a schedule)
  - `tool` → **Automation** category — a reusable utility invoked during workflow execution
  - `agent` → **Reasoning** category — an AI agent that reasons over data or coordinates sub-tasks
</Note>

## List skills

Retrieve all skills in your workspace, ordered by creation date descending. You can filter by `type`, `status`, or a name search string.

```
GET /api/skills
```

### Query parameters

<ParamField query="type" type="string">
  Filter by skill type. Must be one of `action`, `trigger`, `tool`, or `agent`.
</ParamField>

<ParamField query="status" type="string">
  Filter by skill status. Must be `active` or `inactive`.
</ParamField>

<ParamField query="search" type="string">
  Case-insensitive substring search against the skill `name` field.
</ParamField>

<CodeGroup>

```bash curl
# List all skills
curl https://app.atmet.com/api/skills \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>"

# List active agent-type skills
curl "https://app.atmet.com/api/skills?type=agent&status=active" \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>"
```

```javascript JavaScript
const response = await fetch("https://app.atmet.com/api/skills?type=agent&status=active", {
  headers: {
    "Authorization": "Bearer <key>",
    "X-Workspace-Id": "<workspace-id>",
  },
});

const { data } = await response.json();
console.log(data.skills);
```

</CodeGroup>

### Response

```json
{
  "data": {
    "skills": [
      {
        "id": "skl_a1b2c3d4-e5f6-7890-abcd-ef1234567890",
        "name": "Summarise thread",
        "description": "Reads a message thread and produces a concise bullet-point summary.",
        "type": "agent",
        "status": "active",
        "created_by": "usr_01j9x...",
        "created_at": "2025-01-10T09:00:00Z",
        "updated_at": "2025-01-12T14:20:00Z"
      }
    ]
  }
}
```

<ResponseField name="data.skills" type="Skill[]" required>
  Array of Skill objects belonging to the workspace.
</ResponseField>

## Create a skill

Add a new skill to your workspace. You must supply a `name` and `type`. New skills are created with a status of `active`.

```
POST /api/skills
```

### Request body

<ParamField body="name" type="string" required>
  Display name for the skill.
</ParamField>

<ParamField body="description" type="string">
  A short explanation of what the skill does and when it should be used.
</ParamField>

<ParamField body="type" type="string" required>
  The capability category. Must be one of `action`, `trigger`, `tool`, or `agent`.
</ParamField>

<ParamField body="definition" type="object">
  Optional configuration object for the skill. Supply any key-value pairs your skill implementation requires.
</ParamField>

<ParamField body="status" type="string">
  Initial lifecycle status. Must be `active` or `inactive`. Defaults to `active`.
</ParamField>

<CodeGroup>

```bash curl
curl -X POST https://app.atmet.com/api/skills \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Summarise thread",
    "description": "Reads a message thread and produces a concise bullet-point summary.",
    "type": "agent"
  }'
```

```javascript JavaScript
const response = await fetch("https://app.atmet.com/api/skills", {
  method: "POST",
  headers: {
    "Authorization": "Bearer <key>",
    "X-Workspace-Id": "<workspace-id>",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Summarise thread",
    description: "Reads a message thread and produces a concise bullet-point summary.",
    type: "agent",
  }),
});

const { data } = await response.json();
console.log(data.skill);
```

</CodeGroup>

### Response `201`

```json
{
  "data": {
    "skill": {
      "id": "skl_a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "name": "Summarise thread",
      "description": "Reads a message thread and produces a concise bullet-point summary.",
      "type": "agent",
      "status": "active",
      "created_by": "usr_01j9x...",
      "created_at": "2025-01-15T10:30:00Z",
      "updated_at": "2025-01-15T10:30:00Z"
    }
  }
}
```

<ResponseField name="data.skill" type="Skill" required>
  The newly created Skill object.
</ResponseField>

## Update a skill

Modify the name, description, type, definition, or status of an existing skill. All fields are optional; only the fields you include are changed.

```
PATCH /api/skills/{id}
```

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the skill to update.
</ParamField>

### Request body

<ParamField body="name" type="string">
  New display name for the skill.
</ParamField>

<ParamField body="description" type="string">
  Updated description of what the skill does.
</ParamField>

<ParamField body="type" type="string">
  Updated capability category. Must be one of `action`, `trigger`, `tool`, or `agent`.
</ParamField>

<ParamField body="definition" type="object">
  Replacement configuration object for the skill.
</ParamField>

<ParamField body="status" type="string">
  New lifecycle status. Must be `active` or `inactive`. Set to `inactive` to disable the skill without deleting it.
</ParamField>

<CodeGroup>

```bash curl
curl -X PATCH \
  https://app.atmet.com/api/skills/skl_a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>" \
  -H "Content-Type: application/json" \
  -d '{"status": "inactive"}'
```

```javascript JavaScript
const skillId = "skl_a1b2c3d4-e5f6-7890-abcd-ef1234567890";

const response = await fetch(`https://app.atmet.com/api/skills/${skillId}`, {
  method: "PATCH",
  headers: {
    "Authorization": "Bearer <key>",
    "X-Workspace-Id": "<workspace-id>",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ status: "inactive" }),
});

const { data } = await response.json();
console.log(data.skill);
```

</CodeGroup>

### Response

```json
{
  "data": {
    "skill": {
      "id": "skl_a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "name": "Summarise thread",
      "description": "Reads a message thread and produces a concise bullet-point summary.",
      "type": "agent",
      "status": "inactive",
      "created_by": "usr_01j9x...",
      "created_at": "2025-01-10T09:00:00Z",
      "updated_at": "2025-01-15T16:00:00Z"
    }
  }
}
```

<ResponseField name="data.skill" type="Skill" required>
  The updated Skill object.

## Delete a skill

Permanently remove a skill from your workspace. Any chats or automations that reference this skill will lose the link.

```
DELETE /api/skills/{id}
```

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the skill to delete.
</ParamField>

<CodeGroup>

```bash curl
curl -X DELETE \
  https://app.atmet.com/api/skills/skl_a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>"
```

```javascript JavaScript
const skillId = "skl_a1b2c3d4-e5f6-7890-abcd-ef1234567890";

await fetch(`https://app.atmet.com/api/skills/${skillId}`, {
  method: "DELETE",
  headers: {
    "Authorization": "Bearer <key>",
    "X-Workspace-Id": "<workspace-id>",
  },
});
```

</CodeGroup>

### Response

```json
{
  "data": {
    "success": true
  }
}
```

## Attaching skills to chats

You can link any skill to a specific chat so the Atmet assistant has access to it during the conversation. Use the sub-resource endpoints under `/api/chats/{id}/skills`.

### List skills on a chat

```
GET /api/chats/{id}/skills
```

Returns the skills currently attached to the chat.

```json
{
  "data": {
    "skills": [
      {
        "skill": {
          "id": "skl_a1b2c3d4-e5f6-7890-abcd-ef1234567890",
          "name": "Summarise thread",
          "type": "agent",
          "status": "active",
          "description": "Reads a message thread and produces a concise bullet-point summary."
        }
      }
    ]
  }
}
```

### Link a skill to a chat

```
POST /api/chats/{id}/skills
```

<ParamField body="id" type="string" required>
  UUID of the skill to link.
</ParamField>

Returns `201` on success:

```json
{
  "data": {
    "success": true
  }
}
```

Returns `409` if the skill is already linked to the chat.

### Unlink a skill from a chat

```
DELETE /api/chats/{id}/skills
```

<ParamField body="id" type="string" required>
  UUID of the skill to remove from the chat.
</ParamField>

Returns `200` with `{ "data": { "success": true } }`.

## Error responses

All error responses follow the same envelope shape:

```json
{
  "error": {
    "message": "Skill not found."
  }
}
```

| HTTP status | When it occurs |
|---|---|
| `400` | Invalid JSON body or a validation failure |
| `401` | Missing or invalid API key |
| `404` | Skill not found |
| `409` | Skill is already linked to the target chat |
| `500` | Unexpected server error |
