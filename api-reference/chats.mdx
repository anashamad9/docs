The Chats API lets you programmatically manage every AI conversation in your Atmet workspace. You can create a new chat, retrieve existing ones, update metadata like the title or archive status, delete chats you no longer need, and exchange messages with the Atmet AI assistant — all through a consistent REST interface. Every request must carry your API key and workspace identifier in the request headers.

---

## Authentication

All endpoints require the following headers:

| Header | Description |
|---|---|
| `Authorization` | `Bearer <key>` — your Atmet API key |
| `X-Workspace-Id` | The unique identifier of your workspace |

---

## The Chat object

Every endpoint that returns a chat yields the following fields.

<ResponseField name="id" type="string" required>
  Unique identifier for the chat (UUID).
</ResponseField>

<ResponseField name="title" type="string" required>
  Human-readable title. Auto-generated from the first message when not supplied at creation time.
</ResponseField>

<ResponseField name="status" type="string" required>
  Lifecycle state of the chat. One of `active` or `archived`.
</ResponseField>

<ResponseField name="created_by" type="string" required>
  UUID of the user who created the chat.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the chat was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the most recent update (including new messages).
</ResponseField>

## List chats

Retrieve all chats in your workspace, ordered by most recently updated first.

```
GET /api/chats
```

<CodeGroup>

```bash curl
curl https://app.atmet.com/api/chats \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>"
```

```javascript JavaScript
const response = await fetch("https://app.atmet.com/api/chats", {
  headers: {
    "Authorization": "Bearer <key>",
    "X-Workspace-Id": "<workspace-id>",
  },
});

const { data } = await response.json();
console.log(data.chats);
```

</CodeGroup>

### Response

```json
{
  "data": {
    "chats": [
      {
        "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
        "title": "Q3 pipeline review",
        "status": "active",
        "created_by": "usr_01j9x...",
        "created_at": "2025-01-15T10:30:00Z",
        "updated_at": "2025-01-15T11:45:00Z"
      }
    ]
  }
}
```

<ResponseField name="data.chats" type="Chat[]" required>
  Array of Chat objects belonging to the workspace.
</ResponseField>

## Create a chat

Create a new AI conversation. The creator is automatically added as a participant.

```
POST /api/chats
```

### Request body

<ParamField body="title" type="string">
  Display name for the chat. When omitted, Atmet generates a title automatically after the first exchange.
</ParamField>

<CodeGroup>

```bash curl
curl -X POST https://app.atmet.com/api/chats \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>" \
  -H "Content-Type: application/json" \
  -d '{"title": "Launch checklist review"}'
```

```javascript JavaScript
const response = await fetch("https://app.atmet.com/api/chats", {
  method: "POST",
  headers: {
    "Authorization": "Bearer <key>",
    "X-Workspace-Id": "<workspace-id>",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ title: "Launch checklist review" }),
});

const { data } = await response.json();
console.log(data.chat);
```

</CodeGroup>

### Response `201`

```json
{
  "data": {
    "chat": {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "title": "Launch checklist review",
      "status": "active",
      "created_by": "usr_01j9x...",
      "created_at": "2025-01-15T10:30:00Z",
      "updated_at": "2025-01-15T10:30:00Z"
    }
  }
}
```

<ResponseField name="data.chat" type="Chat" required>
  The newly created Chat object.
</ResponseField>

## Get a chat

Retrieve a single chat by its ID. The response also includes linked users, skills, automations, and schedules.

```
GET /api/chats/{id}
```

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the chat to retrieve.
</ParamField>

<CodeGroup>

```bash curl
curl https://app.atmet.com/api/chats/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>"
```

```javascript JavaScript
const chatId = "a1b2c3d4-e5f6-7890-abcd-ef1234567890";

const response = await fetch(`https://app.atmet.com/api/chats/${chatId}`, {
  headers: {
    "Authorization": "Bearer <key>",
    "X-Workspace-Id": "<workspace-id>",
  },
});

const { data } = await response.json();
console.log(data.chat);
```

</CodeGroup>

### Response

```json
{
  "data": {
    "chat": {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "title": "Launch checklist review",
      "status": "active",
      "created_by": "usr_01j9x...",
      "workspace_id": "ws_abc123",
      "created_at": "2025-01-15T10:30:00Z",
      "updated_at": "2025-01-15T11:45:00Z"
    },
    "users": [...],
    "skills": [...],
    "automations": [...],
    "schedules": [...]
  }
}
```

<ResponseField name="data.chat" type="Chat" required>
  The Chat object.
</ResponseField>

<ResponseField name="data.users" type="object[]" required>
  Participants linked to this chat.

  <Expandable title="User fields">
    <ResponseField name="id" type="string">UUID of the user.</ResponseField>
    <ResponseField name="email" type="string">Email address of the user.</ResponseField>
    <ResponseField name="full_name" type="string">Display name of the user.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="data.skills" type="object[]" required>
  Skills attached to this chat. Each entry contains a summary of the linked Skill object (`id`, `name`, `type`, `status`).
</ResponseField>

<ResponseField name="data.automations" type="object[]" required>
  Automations attached to this chat. Each entry contains a summary of the linked Automation object (`id`, `name`, `status`).
</ResponseField>

<ResponseField name="data.schedules" type="object[]" required>
  Schedules attached to this chat. Each entry contains `id`, `name`, `cron_expression`, and `status`.
</ResponseField>

## The Message object

All message endpoints return objects with the following fields.

<ResponseField name="id" type="string" required>
  Unique identifier for the message (UUID).
</ResponseField>

<ResponseField name="role" type="string" required>
  Who authored the message. One of `user`, `assistant`, `system`, or `tool`.
</ResponseField>

<ResponseField name="content" type="string" required>
  Full text content of the message.
</ResponseField>

<ResponseField name="metadata" type="object">
  Optional metadata. For `assistant` messages this includes model usage statistics.

  <Expandable title="Metadata fields (assistant messages)">
    <ResponseField name="model" type="string">Model identifier used to generate the response, e.g. `gpt-4o`.</ResponseField>
    <ResponseField name="prompt_tokens" type="number">Number of tokens in the prompt.</ResponseField>
    <ResponseField name="completion_tokens" type="number">Number of tokens in the completion.</ResponseField>
    <ResponseField name="finish_reason" type="string">Reason the model stopped generating, e.g. `stop`.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the message was created.
</ResponseField>

## List messages

Return all messages in a chat in ascending chronological order. Supports cursor-based pagination using the `before` parameter.

```
GET /api/chats/{id}/messages
```

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the chat whose messages you want to retrieve.
</ParamField>

### Query parameters

<ParamField query="limit" type="number">
  Maximum number of messages to return. Defaults to `50`, capped at `100`.
</ParamField>

<ParamField query="before" type="string">
  ISO 8601 timestamp. When provided, only messages created before this time are returned — useful for loading older history.
</ParamField>

<CodeGroup>

```bash curl
curl "https://app.atmet.com/api/chats/a1b2c3d4-e5f6-7890-abcd-ef1234567890/messages?limit=20" \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>"
```

```javascript JavaScript
const chatId = "a1b2c3d4-e5f6-7890-abcd-ef1234567890";

const response = await fetch(
  `https://app.atmet.com/api/chats/${chatId}/messages?limit=20`,
  {
    headers: {
      "Authorization": "Bearer <key>",
      "X-Workspace-Id": "<workspace-id>",
    },
  }
);

const { data } = await response.json();
console.log(data.messages);
```

</CodeGroup>

### Response

```json
{
  "data": {
    "messages": [
      {
        "id": "msg_abc123",
        "role": "user",
        "content": "Summarise the Q3 results",
        "metadata": null,
        "created_at": "2025-01-15T10:31:00Z"
      },
      {
        "id": "msg_def456",
        "role": "assistant",
        "content": "Here is a summary of Q3 results...",
        "metadata": {
          "model": "gpt-4o",
          "prompt_tokens": 312,
          "completion_tokens": 148,
          "finish_reason": "stop"
        },
        "created_at": "2025-01-15T10:31:05Z"
      }
    ],
    "hasMore": false
  }
}
```

<ResponseField name="data.messages" type="Message[]" required>
  Ordered list of Message objects.
</ResponseField>

<ResponseField name="data.hasMore" type="boolean" required>
  `true` when more messages exist before the oldest message returned. Use the `before` query parameter to page backwards.
</ResponseField>

## Send a message

Send a user message to a chat and receive a streamed AI response. Atmet persists both the user message and the completed assistant reply automatically.

```
POST /api/chats/{id}/messages
```

<Note>
  This endpoint returns a **Server-Sent Events (SSE) stream** (`Content-Type: text/event-stream`). Each event carries a JSON delta (`{"content": "..."}`) and the stream ends with a `[DONE]` sentinel. The full assistant reply is saved to the chat after the stream closes.
</Note>

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the chat to send the message to.
</ParamField>

### Request body

<ParamField body="content" type="string" required>
  Text content of the message to send.
</ParamField>

<ParamField body="role" type="string">
  Role of the sender. Defaults to `user`. Accepted value: `user`.
</ParamField>

<CodeGroup>

```bash curl
curl -X POST \
  https://app.atmet.com/api/chats/a1b2c3d4-e5f6-7890-abcd-ef1234567890/messages \
  -H "Authorization: Bearer <key>" \
  -H "X-Workspace-Id: <workspace-id>" \
  -H "Content-Type: application/json" \
  --no-buffer \
  -d '{"content": "Summarise the Q3 results"}'
```

```javascript JavaScript
const chatId = "a1b2c3d4-e5f6-7890-abcd-ef1234567890";

const response = await fetch(
  `https://app.atmet.com/api/chats/${chatId}/messages`,
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer <key>",
      "X-Workspace-Id": "<workspace-id>",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ content: "Summarise the Q3 results" }),
  }
);

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const lines = decoder.decode(value).split("\n");
  for (const line of lines) {
    if (line.startsWith("data: ") && line !== "data: [DONE]") {
      const { content } = JSON.parse(line.slice(6));
      process.stdout.write(content);
    }
  }
}
```

</CodeGroup>

### Response stream

```
data: {"content":"Here"}
data: {"content":" is"}
data: {"content":" a summary..."}
data: [DONE]
```

## Error responses

All error responses follow the same envelope shape:

```json
{
  "error": {
    "message": "Chat not found."
  }
}
```

| HTTP status | When it occurs |
|---|---|
| `400` | Invalid JSON body or a validation failure |
| `401` | Missing or invalid API key |
| `404` | Chat or resource not found |
| `500` | Unexpected server error |
