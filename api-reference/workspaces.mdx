The Workspaces API gives you full programmatic control over workspaces and their membership. You can list every workspace your account belongs to, create new ones, update settings like name and plan, and remove workspaces you own. The members sub-resource lets you inspect who has access to a workspace, invite new members by email, update their roles, and remove them when necessary. All write operations on a workspace require you to be either the owner or an admin-level member.

---

## The Workspace object

<ResponseField name="id" type="string" required>
  Unique identifier for the workspace (UUID).
</ResponseField>

<ResponseField name="name" type="string" required>
  Display name of the workspace.
</ResponseField>

<ResponseField name="plan" type="string" required>
  Billing plan for the workspace. One of `free`, `pro`, or `enterprise`.
</ResponseField>

<ResponseField name="owner_id" type="string" required>
  User ID of the workspace owner.
</ResponseField>

<ResponseField name="role" type="string">
  The authenticated user's role in this workspace. Either `owner`, `admin`, or `member`. **Only present in the list workspaces response** — not returned when fetching a single workspace by ID.
</ResponseField>

<ResponseField name="avatar_url" type="string | null">
  Absolute URL of the workspace avatar image, or `null` if none is set.
</ResponseField>

<ResponseField name="country" type="string | null">
  ISO 3166-1 alpha-2 country code associated with the workspace (e.g. `US`, `DE`), or `null`.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the workspace was created.
</ResponseField>

## List workspaces

Returns all workspaces where your account has an active membership, including the role you hold in each one.

```
GET /api/workspaces
```

**Headers**

| Header | Required | Description |
|---|---|---|
| `Authorization` | Yes | `Bearer <key>` — your Atmet API key |

**Response**

```json
{
  "data": {
    "workspaces": [Workspace]
  }
}
```

<CodeGroup>
```bash curl
curl https://atmet.ai/api/workspaces \
  -H "Authorization: Bearer atmet_your_api_key"
```

```javascript JavaScript
const response = await fetch("https://atmet.ai/api/workspaces", {
  headers: {
    "Authorization": "Bearer atmet_your_api_key",
  },
});

const { data } = await response.json();
console.log(data.workspaces);
```
</CodeGroup>

**Example response**

```json
{
  "data": {
    "workspaces": [
      {
        "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
        "name": "Acme Corp",
        "plan": "pro",
        "owner_id": "user_abc123",
        "role": "owner",
        "avatar_url": "https://example.com/avatars/acme.png",
        "country": "US",
        "created_at": "2024-09-15T10:00:00.000Z"
      }
    ]
  }
}
```

## Create a workspace

Creates a new workspace and automatically adds you as its owner.

```
POST /api/workspaces
```

**Headers**

| Header | Required | Description |
|---|---|---|
| `Authorization` | Yes | `Bearer <key>` — your Atmet API key |
| `Content-Type` | Yes | `application/json` |

**Request body**

<ParamField body="name" type="string" required>
  The display name for the new workspace. Must be a non-empty string.
</ParamField>

<ParamField body="plan" type="string">
  Billing plan for the workspace. One of `free`, `pro`, or `enterprise`. Defaults to `free` when omitted.
</ParamField>

<ParamField body="avatar_url" type="string">
  Absolute URL for the workspace avatar image. Must be a valid URL. Optional.
</ParamField>

**Response** — `201 Created`

```json
{
  "data": {
    "workspace": Workspace
  }
}
```

<CodeGroup>
```bash curl
curl -X POST https://atmet.ai/api/workspaces \
  -H "Authorization: Bearer atmet_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Acme Corp",
    "plan": "pro"
  }'
```

```javascript JavaScript
const response = await fetch("https://atmet.ai/api/workspaces", {
  method: "POST",
  headers: {
    "Authorization": "Bearer atmet_your_api_key",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Acme Corp",
    plan: "pro",
  }),
});

const { data } = await response.json();
console.log(data.workspace.id);
```
</CodeGroup>

## Delete a workspace

Permanently deletes a workspace. Only the workspace **owner** can perform this action — admin members receive `403 Forbidden`.

<Warning>
  **This action is irreversible.** Deleting a workspace permanently removes all automations, skills, integration connections, and member associations tied to it. There is no recovery path. Make sure you have exported or migrated anything you want to keep before calling this endpoint.
</Warning>

```
DELETE /api/workspaces/{id}
```

**Path parameters**

<ParamField path="id" type="string" required>
  The UUID of the workspace to delete.
</ParamField>

**Headers**

| Header | Required | Description |
|---|---|---|
| `Authorization` | Yes | `Bearer <key>` — your Atmet API key |

**Response**

```json
{
  "data": {
    "success": true
  }
}
```

<CodeGroup>
```bash curl
curl -X DELETE https://atmet.ai/api/workspaces/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "Authorization: Bearer atmet_your_api_key"
```

```javascript JavaScript
const workspaceId = "a1b2c3d4-e5f6-7890-abcd-ef1234567890";

const response = await fetch(`https://atmet.ai/api/workspaces/${workspaceId}`, {
  method: "DELETE",
  headers: {
    "Authorization": "Bearer atmet_your_api_key",
  },
});

const { data } = await response.json();
console.log(data.success); // true
```
</CodeGroup>

## Invite a workspace member

Sends an email invitation to a user to join the workspace. You must be the workspace owner or an admin to call this endpoint. If the invited email already has a pending invitation for this workspace, the request returns `409 Conflict`. The endpoint also enforces the workspace seat limit — if the number of active members plus pending invitations would exceed it, the request returns `400 Bad Request`.

```
POST /api/workspaces/{id}/members
```

**Path parameters**

<ParamField path="id" type="string" required>
  The UUID of the workspace to invite the member into.
</ParamField>

**Headers**

| Header | Required | Description |
|---|---|---|
| `Authorization` | Yes | `Bearer <key>` — your Atmet API key |
| `Content-Type` | Yes | `application/json` |

**Request body**

<ParamField body="email" type="string" required>
  The email address of the person to invite. Must be a valid email format.
</ParamField>

<ParamField body="role" type="string">
  The role to assign once the invitation is accepted. Currently only `"member"` is accepted. Defaults to `"member"` when omitted.
</ParamField>

**Response** — `201 Created`

<ResponseField name="invitation" type="object" required>
  The created invitation record.
  <Expandable title="invitation fields">
    <ResponseField name="id" type="string" required>
      Unique identifier for the invitation (UUID).
    </ResponseField>
    <ResponseField name="workspace_id" type="string" required>
      The workspace the invitation belongs to.
    </ResponseField>
    <ResponseField name="email" type="string" required>
      The email address the invitation was sent to.
    </ResponseField>
    <ResponseField name="role" type="string" required>
      The role the invitee will receive on acceptance.
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Invitation status. `pending` immediately after creation.
    </ResponseField>
    <ResponseField name="token" type="string" required>
      The accept token embedded in the invitation email link.
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of when the invitation was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="invitationEmailSent" type="boolean" required>
  `true` if the invitation email was delivered successfully.
</ResponseField>

<ResponseField name="invitationEmailWarning" type="string | null" required>
  `null` when the email was sent. Contains a human-readable reason string if delivery failed (the invitation record is still created).
</ResponseField>

```json
{
  "data": {
    "invitation": {
      "id": "inv_a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "workspace_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "email": "charlie@acme.com",
      "role": "member",
      "status": "pending",
      "token": "eyJhbGciOiJIUzI1NiJ9...",
      "created_at": "2024-12-10T11:00:00.000Z"
    },
    "invitationEmailSent": true,
    "invitationEmailWarning": null
  }
}
```

<CodeGroup>
```bash curl
curl -X POST https://atmet.ai/api/workspaces/a1b2c3d4-e5f6-7890-abcd-ef1234567890/members \
  -H "Authorization: Bearer atmet_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "charlie@acme.com",
    "role": "member"
  }'
```

```javascript JavaScript

const response = await fetch(
  `https://atmet.ai/api/workspaces/${workspaceId}/members`,
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer atmet_your_api_key",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      email: "charlie@acme.com",
      role: "member",
    }),
  }
```

## Error responses

All errors follow the standard Atmet error envelope.

```json
{
  "error": {
    "message": "Workspace not found."
  }
}
```

| HTTP status | When it occurs |
|---|---|
| `400 Bad Request` | Invalid request body, attempting to remove the workspace owner, or seat limit reached |
| `401 Unauthorized` | Missing or invalid Bearer token |
| `403 Forbidden` | Valid token but you are not the owner or an admin (for write operations) |
| `404 Not Found` | The workspace or member ID does not exist, or you are not a member |
| `409 Conflict` | A pending invitation for the given email already exists in this workspace |
| `500 Internal Server Error` | Unexpected server-side failure |
